GLPI

Départage GLPI pour les projets SuperCar-Web & VMS - Gestion de parc et ticketing adaptés aux contextes professionnels

1. Installation & Configuration

SuperCar-Web

  • Installation de GLPI sur serveur Ubuntu 12.04.2 LTS (virtualisé)
  • Configuration Apache/MySQL pour cohabitation avec l'application web PHP
  • Intégration avec l'infrastructure existante (4 entrepôts + siège social)
  • Paramétrage des catégories : Développement Web, Hébergement, Base de données, Administration du site

VMS (VoucherManager)

  • Déploiement GLPI sur serveur PostgreSQL centralisé
  • Configuration multi-sites (siège + points de vente/magasins)
  • Mise en place de connexions ODBC pour reporting Excel
  • Catégories spécifiques : Application JavaFX (siège), Application magasin, Base de données, Génération/Envoi bons, Rédemption

2. Gestion des Tickets

SuperCar-Web

Types de tickets courants :

  • Bugs sur le formulaire de demande d'essai
  • Problèmes d'affichage responsive (desktop/tablette/smartphone)
  • Erreurs dans le module d'administration
  • Dysfonctionnements de la galerie avec pagination
  • Incidents sur l'hébergement distant

Workflow : Utilisateurs : Admin site, Équipe développement MultiSys, Vendeurs (salesmen). Priorités selon impact sur visibilité web et génération de leads.

VMS (VoucherManager)

Types de tickets courants :

  • Échec de génération de bons PDF (QR/Code-barres)
  • Problèmes d'envoi email automatique
  • Erreurs de validation lors de la rédemption en magasin
  • Bugs dans le processus paiement/approbation
  • Incidents de scan QR en point de vente

Workflow : Utilisateurs : Admin/Comptable (siège), Approbateurs, Superviseurs magasin, Équipe SSII. Priorités critiques pour les blocages de rédemption (impact direct sur clients).

3. Gestion de Parc

SuperCar-Web

Inventaire matériel :

  • Serveur physique (Intel Core i5, 8GB RAM, 1TB HDD)
  • Postes de travail développeurs (équipes MultiSys)
  • Smartphones pour vendeurs (accès mobile au site)

Inventaire logiciel :

  • Licences : Visual Studio 2015/2017/2019, PHPStorm
  • Serveurs : Xampp/Wamp, Apache, MySQL
  • Frameworks : Bootstrap, jQuery
  • Outils : FileZilla, phpMyAdmin

VMS (VoucherManager)

Inventaire matériel :

  • Serveur PostgreSQL centralisé
  • Postes siège (application JavaFX complète)
  • Postes magasins (application légère + scanners QR)

Inventaire logiciel :

  • JDK/JavaFX (multi-utilisateur)
  • PostgreSQL + pgAdmin
  • Bibliothèques QR/Barcode
  • Connecteur ODBC pour Excel
  • Serveur SMTP (emails automatiques)

4. Gestion des Utilisateurs

SuperCar-Web

Profils dans GLPI :

  • Chef de projet (rotation) : Vision globale, attribution tickets
  • Développeurs web : Tickets HTML/CSS/PHP/MySQL
  • Administrateur SuperCar : Tickets module admin, contenu dynamique
  • Support technique : Incidents hébergement, serveur
  • Vendeurs : Déclaration bugs fonctionnels (demandes d'essai)

VMS (VoucherManager)

Profils dans GLPI :

  • Admin/Comptable siège : Tickets saisie demandes, facturation
  • Approbateurs : Tickets workflow validation
  • Développeurs JavaFX : Tickets techniques application
  • Superviseurs magasin : Tickets rédemption, scan QR
  • DBA PostgreSQL : Tickets base de données, procédures/triggers
  • Support SSII : Coordination générale

5. Rapports & Statistiques

SuperCar-Web

Indicateurs clés :

  • Nombre de tickets par module (Galerie, Demande essai, Admin, Contact)
  • Temps de résolution moyen par type (bugs responsive vs. bugs backend)
  • Disponibilité de l'hébergement en ligne
  • Tickets bloquants pour mise en production
  • Rotation chefs de projet : tickets traités par équipe

Rapports périodiques : Bilan hebdomadaire des développements (avancement cahier des charges), suivi compatibilité multi-écrans (résolution 1024×768 minimum), conformité aux exigences (procédures stockées, triggers, design responsiveness).

VMS (VoucherManager)

Indicateurs clés :

  • Tickets critiques bloquant émission/rédemption de bons
  • Performance de génération PDF (temps de traitement)
  • Incidents sur envois emails automatiques
  • Taux d'échec de validation QR en magasin
  • Disponibilité du serveur central (impact multi-sites)

Rapports périodiques : Audit trail (traçabilité des interventions techniques), suivi des SLA (délais d'émission, réactivité support), statistiques par site (siège vs. magasins), conformité sécurité (anti-fraude, unicité codes), rapports ODBC Excel (corrélation avec incidents techniques).
